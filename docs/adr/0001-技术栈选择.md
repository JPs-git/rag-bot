# ADR 0001: 技术栈选择

## 决策背景

本项目是一个基于文档RAG的问答助手，作为求职演示项目。核心需求：
- 纯前端实现，无需后端
- 方便部署在线演示
- 支持文档上传、分块、向量化、检索
- 提供Ground Truth展示

## 技术决策

### 1. 前端框架
- **选择**: React + Vite
- **理由**: 现代主流框架，生态成熟，Vite构建快速，部署简单

### 2. 样式方案
- **选择**: Tailwind CSS 3 + shadcn/ui + shadcn/ai
- **理由**: 
  - Tailwind CSS 提供快速开发体验
  - shadcn/ui 提供高质量基础组件
  - shadcn/ai 提供专门针对LLM对话的组件（Message、Conversation、Loader等）

### 3. 向量化方案
- **选择**: Transformers.js + Xenova/multilingual-e5-small
- **理由**: 
  - 支持在浏览器中运行开源embedding模型
  - multilingual-e5-small支持中文，模型大小适中（470MB）

### 4. 向量存储
- **选择**: 内存存储（Session级别）
- **理由**: 符合用户需求，刷新页面后重新上传，实现简单

### 5. 状态管理
- **选择**: React Context + useReducer
- **理由**: 内置方案，无需额外依赖，适合当前复杂度

### 6. 配置持久化
- **选择**: localStorage
- **理由**: 方便开发和用户体验，API Key等配置自动保存

### 7. 分块策略
- **选择**: 策略模式 + 递归字符分块
- **理由**: 可扩展性强，递归字符分块效果好

### 8. 检索策略
- **选择**: 策略模式 + 余弦相似度 + Top-K
- **理由**: 实现简单，效果良好，便于扩展

## 预期影响

- 纯前端架构，部署简单（可直接部署到Vercel/Netlify）
- 无后端依赖，成本低
- 会话级别数据，隐私安全
- 策略模式支持未来扩展其他分块/检索算法

## 备选方案评估

| 方案 | 优点 | 缺点 |
|------|------|------|
| Next.js | 服务端能力强 | 增加复杂度 |
| Zustand | 轻量状态管理 | 额外依赖 |
| WebLLM | 完全本地运行 | 性能差，模型选择有限 |
